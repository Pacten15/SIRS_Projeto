For Authenticity:

AES-192 for encryption and decryption


For Intregaty:

hash function -> SHA-2

Execute hash on the client side and afterwards the server does the same and verifies what was received to see if the have the same hash which means the file wasn´t modified during the transmission of it. 


For Confidentiality:


Server encrypts message using public key of the user






1º Client:

-Gera um número (por intermédio do random)

-junta a mensagem com o número

-encripta a mensagem inteira usando RSA usando a chave publica do servidor

-enviar ao servidor 


2º Server:

-Recebe a mensagem e desincripta com a sua chave privada do server (confidentiality)

-Encripta o resultado usando uma digital signature (Autheticity) e junta a hash do resultado e o número, usando a chave pública do user

-Envia ao client usando SSL


3º Client 

-Recebe o resultado e desencripta com a chave privada dele e finalmente consegue ver o resultado e antes de o ver executa a hash nesse resultado para verificar se hash recebida pela menssagem é a mesma hash obtida pelo o cliente.
